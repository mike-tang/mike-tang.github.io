Demandforce Learn

HTML5, CSS3, JS, jQuery, Interaction Design, Responsive Design

www.demandforce.com/learn

Lead designer and developer for Demandforce Learn, an offline-enabled webpage optimized for tablet usage.

The main purpose of this page was to serve as a quick product overview for sales representatives and partners to use while traveling and speaking with potential customers. Since internet connectivity isn't always guaranteed at tradeshows or at various small business locations, DF reps needed an offline alternative to our website.

To solve this problem, I designed and built a lightweight HTML5 webpage with smooth CSS3 transitions.

The wireframes were relatively simple, so I skipped drafting mockups and went straight into developing and iterating on a prototype that would eventually launch as the live page.

Without a doubt, this webpage needed to be fluid responsive so that it could cater to all types of mobile devices. I built a custom grid that responded to landscape and portrait views on tablets and phones, and used FitVids.js to make sure videos resized properly as well. To further improve performance and flexibility, I built an icon font with IcoMoon. Finally, I enabled offline access by listing all assets in the cache manifest, which tells the browser what to store in the application cache. The total size of the page (minus the video, which can’t be cached) came out to 643kb - just enough for a busy Demandforce representative to download within a few seconds in the office and head out to her next event.

Fluid Responsive Email Templates

HTML, CSS, Responsive Design, universal compatibility

Download

Lead developer for all Demandforce marketing email communications. I created fluid responsive email templates to improve the user experience of our communications across all devices, browsers, email clients, and mobile apps.

Coding emails that work everywhere can be tricky. Devices come in various sizes, browsers understand code differently, and email clients and apps provide varying degrees of support for HTML elements and CSS styling. There was a time when media queries would suffice to get responsive emails working on native Android and iOS mail apps, but their support across different email clients and apps is limited. Fortunately, there is a solution for this - fluid responsive grids.

Fluid grids work in emails by creating nested table layouts with percentage widths rather than pixels. This method allows all text, images, and buttons to adjust according to device size.

A maximum width is set on the parent table to restrict content from expanding too wide on the desktop view. Microsoft Outlook 2007/2010/2013 versions don't support max-width, but conditional comments for Outlook can be used to circumvent this issue.

The Journey

Post undergrad, I was on track to pursue a career in sustainable urban design. But just as I was about to submit grad school apps, I was hit with a huge epiphany - if I was going to make any meaningful change in this world, it wouldn't be through local-scale projects. It'd have to be through design and technology that could be easily scaled and distributed globally.

At this point, I went all in with the career pivot. My goal was to end up in a role where I'd be at the intersection of design, code, and an idea that could indiscriminately improve human quality of life. To get started, I learned the basics of web technology on Codecademy and Treehouse. I was also lucky enough to stumble upon Coursera's Human Computer Interaction course and gain a healthy boost of knowledge in user experience design principles. Passion and grit took care of the rest.
